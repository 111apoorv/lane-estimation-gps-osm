PROJECT BACKGROUND & CONTEXT
================================================================================

PROBLEM STATEMENT:
OpenStreetMap (OSM) is a free, collaborative map of the world used by millions of applications for navigation, urban planning, and traffic simulation. However, OSM has a critical limitation: many roads are missing accurate lane count information. This missing data affects:
- GPS navigation accuracy
- Traffic simulation models
- Urban planning decisions
- Autonomous vehicle mapping

OUR SOLUTION:
We developed a machine learning model that predicts the number of lanes on roads using GPS probe data from vehicles. When cars drive on roads, their GPS traces create distinct patterns based on which lane they're traveling in. Our model learns these patterns to predict lane counts.

WHAT IS AADT?
  AADT = Annual Average Daily Traffic
  Definition: The average number of vehicles that pass a point on a road 
              in a 24-hour period, averaged over a full year.
  Example: AADT of 50,000 means ~50,000 cars drive on that road segment per day.

METHODOLOGY & MODELS TESTED
================================================================================

APPROACH:
The core insight is that GPS traces from vehicles form distinct clusters/peaks corresponding to different lanes. By analyzing the lateral distribution of GPS points across a road segment, we can infer how many lanes exist.

MODELS TESTED:

1. RANDOM FOREST CLASSIFIER (Best Performer)
   - Parameters: n_estimators=200, class_weight='balanced_subsample'
   - Features: 40 GPS bins + oneway flag = 41 features
   - Result: 90.11% accuracy

2. XGBOOST CLASSIFIER
   - Parameters: objective='multi:softprob', eta=0.07, max_depth=8
   - Result: ~85% accuracy

3. GAUSSIAN MIXTURE MODEL (Unsupervised Baseline)
   - Approach: Detect number of peaks in GPS distribution
   - Result: 22% accuracy (not effective)

4. LIGHTGBM CLASSIFIER
   - Parameters: n_estimators=500, learning_rate=0.05
   - Result: 84.56% accuracy

================================================================================
KEY FINDINGS FROM ERROR ANALYSIS
================================================================================

FINDING 1: Model Struggles with 1-Lane and 3-Lane Roads
- 1-lane roads: 27.5% error rate
- 3-lane roads: 27.4% error rate
- 2-lane roads: Only 2.8% error rate (best)
- Reason: 1-lane and 3-lane roads have less distinct GPS distribution patterns

FINDING 2: Adjacent Lane Counts Are Easily Confused
- Most errors are predictions off by just 1 lane
- 2↔3 lane confusion is the most common mistake
- The GPS patterns for adjacent lane counts are similar

FINDING 3: One-Way vs Two-Way Roads
- 71.9% of "off by 1 lane" errors occur on one-way roads
- One-way roads should have cleaner patterns but show high error
- This suggests one-way flag alone isn't sufficient

FINDING 4: GPS Distribution Spread
- Incorrect predictions have slightly lower spread (std dev: 0.0349-0.0365)
- Correct predictions have higher spread (std dev: 0.0370)
- More uniform distribution → harder to distinguish lanes
